# Estimation Rules

1. Prefer ranges.
2. Name the method.
3. List assumptions.
4. Avoid double-counting.
5. Separate observed from inferred intervals.
6. State exclusions.
7. Never imply stopwatch accuracy.

## Active-session heuristic

Sort reliable events by time. Group events when gaps are 30 minutes or less. A one-event session has no duration unless the event itself has one. For multi-event sessions, use first-to-last time as an observed span. Add no time before the first or after the last event. A broader range may add up to 15 minutes around dense sessions if clearly labeled as an assumption.

## AI activity

Report prompt and response counts, timestamped session spans, requested task types, and whether generated material appears in final artifacts. Do not compare AI latency with human labor.

## Percentages

Avoid a single attribution percentage. If explicitly required, use only a narrow measurable unit such as identical final lines, documented events, or tracked time, and explain that it does not represent total creative contribution.
